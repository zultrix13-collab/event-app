## 5. НАРИЙН ТӨВӨГТЭЙ ХЭСГҮҮДИЙН ЗАДАРГАА

> Доорх хэсгүүд нь хэрэгжүүлэхэд онцгой анхаарал, техникийн гүнзгий мэдлэг шаарддаг. Эхний спринтэд proof-of-concept (PoC) хийж баталгаажулахыг зөвлөж байна.

### 5.1 QR + NFC Дижитал Үнэмлэх (AUTH-03)

**Яагаад төвөгтэй вэ?**
- Интернэтгүй (offline) горимд ч ажиллах ёстой
- NFC hardware дэмжлэг платформ тус бүрт өөр өөр API ашиглана
- QR payload-ийг шифрлэж, expire хугацаатай болгох шаардлагатай
- Хуурамчлах (forgery) эсрэг хамгаалалт — HMAC signature шаардлагатай

**Санал болгох шийдэл:**
- Payload бүтэц: `{ userId, role, eventId, issuedAt, expiresAt, hmac_signature }`
- QR: Dynamic QR — 15 минут бүр refresh (rolling token)
- NFC: NDEF формат ашиглан iOS Core NFC / Android NFC API
- Offline verification: Signature key нь аппд encrypted хэлбэрээр хадгалагдана

### 5.2 Нэгэн Зэрэг Хэрэглэгчийн Ачаалал — Суудал Захиалга (PROG-02)

**Яагаад төвөгтэй вэ?**
- 10,000+ хэрэглэгч нэгэн зэрэг нэг арга хэмжэнд бүртгүүлэхийг оролдвол race condition үүснэ
- Давхар бүртгэлийг (double-booking) сервер түвшинд хаах хэрэгтэй

**Санал болгох шийдэл:**
- Distributed lock: Redis-ийн SETNX эсвэл Redlock алгоритм ашиглах
- Optimistic locking: Database version field ашиглах
- Queue-based: Bull/BullMQ ашиглан захиалгыг дараалуулан боловсруулах
- Capacity counter: Redis atomic increment ашиглан бодит цагт тоолох

### 5.3 AI Chatbot — RAG Архитектур (AI-01, AI-02)

**Яагаад төвөгтэй вэ?**
- Баримт бичиг, хөтөлбөр, FAQ өдөр бүр шинэчлэгдэж болно
- Монгол хэлний NLP чадвар бүхий LLM сонголт хязгаарлагдмал
- Хариулт нь үнэн зөв, hallucination-гүй байх шаардлагатай

**Санал болгох шийдэл:**
- Vector Database: Pinecone / Weaviate / Qdrant — баримт бичгийг embedding хэлбэрт хувиргах
- LLM сонголт: GPT-4o / Claude 3 / Gemini — Монгол хэлний чадвараар үнэлэх
- Retrieval: Semantic search + BM25 hybrid search
- CMS-тэй нийлүүлэлт: Контент шинэчлэгдэх бүрт vector DB-д re-index хийх
- Hallucination хаах: Retrieved context-д суурилсан хариулт — source citation харуулах

### 5.4 Indoor Navigation — Beacon/QR (MAP-04)

**Яагаад төвөгтэй вэ?**
- Дотор орчинд GPS ажиллахгүй — Bluetooth beacon эсвэл QR checkpoint hardware шаардлагатай
- Beacon-ийн физик суурилуулалт, тохиргоо тусдаа ажил
- BLE сигналын хүч хувьсамтгай — байршлын нарийвчлал хязгаарлагдмал

**Санал болгох шийдэл:**
- QR Checkpoint (Хамгийн боломжийн): Байр бүрт QR код байрлуулж scan хийхэд байршил шинэчлэгдэнэ
- BLE Beacon: Bluetooth beacon-ийн RSSI утгаас trilateral байршил тодорхойлох
- Hybrid: QR checkpoint + BLE beacon — нарийвчлалыг нэмэгдүүлэх
- Indoor floor plan: SVG/vector зургийг аппд оруулж байршлын цэгийг overlay хийх

### 5.5 Offline Mode — Гар Утасны Аппликейшн

**Шаардлага:** Хөтөлбөр, газрын зураг, FAQ, яаралтай холбоо интернэтгүйд ажиллах

**Санал болгох шийдэл:**
- SQLite / Realm: Аппийн локал database — sync хийх стратегитай
- Service Worker (веб): PWA технологи ашиглан кэш хийх
- Background sync: Интернэт сэргэхэд автоматаар сервертэй нийлүүлэх
- Conflict resolution: Last-write-wins эсвэл server-wins стратеги

### 5.6 Төлбөрийн Систем (PAY-01 ~ PAY-04)

**Яагаад төвөгтэй вэ?**
- Монгол банкуудын QPay, SocialPay API-тай нэгтгэх
- iOS App Store / Google Play-ийн in-app purchase дүрэм журам дагах
- Transaction idempotency — давхар цэнэглэлтэнд сэргийлэх

**Санал болгох шийдэл:**
- Wallet transaction: ACID transaction — database rollback дэмжих
- Idempotency key: Хүсэлт бүрд өвөрмөц key — давхар боловсруулалтыг хаах
- Payment webhook: Банкны callback-ийг retry механизмтайгаар боловсруулах
- iOS: StoreKit 2 API, Android: Google Play Billing Library

## 7. КОНТЕНТ УДИРДЛАГЫН СИСТЕМ (CMS)

**CMS нь техникийн мэдлэг шаардахгүй бөгөөд зохион байгуулагчид хөтөлбөр, мэдээ, зарлал, илтгэгчийн мэдэээллийг бодит цагт шинэчилж чадна.**

- WYSIWYG editor — rich text засварлах
- Медиа сан — зураг, видео upload, хэмжээг автоматаар оновчлох
- Хянах шатлал — зохиогч → редактор → нийтлэгч
- Олон хэл — Монгол, Англи контентыг зэрэгцүүлэн удирдах
- Preview — нийтлэхийн өмнө харуулах
- Version history — бүх өөрчлөлтийн бүртгэлтэй
- Scheduled publish — тодорхой цагт автоматаар нийтлэх

> Headless CMS хандлага (Strapi, Contentful, Sanity) ашиглах нь веб болон аппликейшнд нэгэн зэрэг агуулга нийлүүлэхэд хамгийн тохиромжтой.
